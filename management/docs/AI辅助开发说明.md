# AI 辅助开发说明

## 使用的 AI 工具
使用 Codex / ChatGPT 类 AI 编程助手辅助完成需求拆解、数据库表设计、接口设计、页面结构、文档初稿和测试排查。

## AI 参与的开发环节
- 需求分析：根据课程评分标准提炼图书管理系统核心模块。
- 数据库设计：生成用户、分类、图书、借阅记录等表结构。
- 代码生成：辅助生成 Controller、Service、Mapper、实体类和前端页面。
- 文档生成：辅助生成 README、需求分析和使用手册。
- Bug 排查：检查接口命名、库存扣减、事务和构建问题。

## 典型提示词示例
1. “根据 Web 高级程序设计实训评分标准设计图书管理系统功能模块。”
2. “生成 Spring Boot + MyBatis 的图书借阅审批接口。”
3. “为图书管理系统设计 MySQL 初始化脚本和测试账号。”
4. “实现一个不依赖外部 API 的模拟 AI 图书推荐模块。”
5. “编写适合课堂验收的 README 和使用手册。”

## AI 生成内容说明
AI 辅助生成了部分 Java 代码、SQL 脚本、前端页面结构和文档初稿。

## 人工修改说明
人工根据项目原有包名、接口返回结构和 Maven 配置进行了整合；补充了库存边界校验、重复借阅判断、事务注解、演示数据和中文界面样式；删除了不符合图书业务的课程描述。

## 测试与验证
通过静态检查确认新增文件和接口结构完整；尝试运行 Maven 测试时，当前环境访问 Maven Central 返回 403，属于网络仓库限制，需要在可访问 Maven 仓库的环境中执行完整构建。

## 反思
AI 能显著提升样板代码和文档产出效率，但生成结果需要人工审查，尤其是数据库约束、事务一致性、异常提示和课程验收场景一致性。
