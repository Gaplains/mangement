## 如果老师要求“真实权限拦截”
当前项目为课堂演示版，前端可切换演示用户，后端保留 Session 登录。若要求严格权限控制，建议：
1. 登录成功后签发 JWT 或保持 Session Cookie；
2. 新增 Spring MVC Interceptor，拦截 `/library/books` 的写操作、`/library/categories/**`、`/library/borrow/*/approve|reject|return`、`/users/**`；
3. 判断当前登录用户角色必须为 `ADMIN`；
4. 读者只能访问自己的借阅记录。

## 如果老师要求“真实大模型 AI”
当前项目已实现可演示的模拟 AI。若要求真实大模型，建议：
1. 在配置文件中增加 `ai.provider`、`ai.api-key`、`ai.base-url`；
2. 在 `AiAssistantServiceImpl` 中把规则逻辑替换为 HTTP API 调用；
3. Prompt 中注入图书分类、图书简介和借阅规则；
4. 保留当前模拟逻辑作为 API 失败时的兜底方案。
